# Dev container — run the local demo in Codespaces

This dev container makes the **local kind demo** of eks-gitops-platform runnable with zero host setup: open the repo in [GitHub Codespaces](https://github.com/features/codespaces) (or any Dev Containers host) and you get a box with **Docker-in-Docker, kind, kubectl, and helm** already installed. After it builds, `postCreate` checks the toolchain and prints the next step — run **`make demo-up`** to build `ghcr.io/charanvamsy26/demo-api:local`, load it into a local `eks-gitops-platform-local` kind cluster (Kubernetes 1.30, no AWS), and install Gatekeeper, kube-prometheus-stack, and the demo-api Helm chart with the cloud-only bits disabled. Ports **8000** (demo-api → http://localhost:8000), **3000** (Grafana, `admin`/`admin`, local-only), and **9090** (Prometheus) are auto-forwarded and labeled in the Ports panel, so once you `kubectl port-forward` (the way `make demo-up` documents) you can open the UIs straight from your browser; tear everything down with `make demo-down`. See [`local/README.md`](../local/README.md) for the full walkthrough.
